# Contributing to GOTM Documentation

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-improvement`)
3. Make your changes to `source/GOTM_Complete_Documentation.md`
4. Run the split script: `python3 source/split_documentation.py`
5. Test locally: `hugo server -D`
6. Commit your changes
7. Push to your fork
8. Submit a pull request

## Guidelines

- **Edit the source:** Always edit `source/GOTM_Complete_Documentation.md`, not files in `content/`
- **Test locally:** Run `hugo server -D` before submitting
- **Clear commits:** Write descriptive commit messages
- **One feature per PR:** Keep pull requests focused

## Questions?

Open an issue or discussion on GitHub.
